Fable 5 是什麼？

Mythos 級模型——能力高於 Opus 4.8，是目前一般使用者能取得的最強 Claude。在 Cursor 的 CursorBench 實戰編碼測試中排名第一。

與 Mythos 5 的差別：兩者底層權重相同，Fable 5 多了一層安全分類器。約 95% 以上的對話不會觸發；觸發時 Cursor 會自動降級到 Opus 4.8，工作不中斷。

五大核心能力

🤖

長程自主代理

在數百萬 tokens 的長對話中維持意圖，減少中途確認。Stripe 實測：5000 萬行 codebase 的全庫遷移，從兩個月壓縮到一天。

⚙️

軟體工程

自行寫測試驗證、多步驟 agent 工作流（搜尋→修改→測試→修復）完成率明顯高於 Opus。

👁️

視覺能力

從設計稿截圖還原前端、讀取圖表數字、用視覺比對實作結果與設計目標。

🧠

深度推理

透過 effort 參數控制思考深度。max 等級會反思並驗證自己的輸出，適合高風險任務。

📄

知識工作

文件推理、財務分析、法律審閱等場景表現突出，適合文件密集的專業工作。

在 Cursor 中啟用

啟用步驟

  1. 前往 Cursor Dashboard → Team Settings
  2. 啟用 Fable 5 並同意資料保留政策
  3. 在模型選單選擇 Claude Fable 5（建議 high thinking 變體）

隱私注意

若帳號開啟 Privacy Mode，需額外同意 Anthropic 的 30 天資料保留條款。Enterprise 需在組織層級核准。啟用 Fable 5 不會改變 Cursor 自身的資料保留設定。

自動降級機制

觸及網路安全、生物化學、模型蒸餾等敏感主題時，Cursor 會靜默切換到 Opus 4.8，不需手動重試。平均不到 5% 的對話會觸發。

計費方式

Fable 5 從 API usage pool 計費（非 fast requests 池）。善用 prompt caching 可享快取讀取 90% 折扣。

針對 morris 專案的建議

目前專案是單頁 HTML/CSS 個人作品集（App Brewery 教學範本）。日常小改動不需要 Fable 5，但以下升級方向值得動用：

推薦

方案 A：設計驅動全面改版

貼 Figma 或參考網站截圖，讓 Fable 5 用 vision 還原設計並迭代修正。

參考這張設計稿，將 Morris Li 個人頁改版為現代化單頁。要求：Tailwind CSS、響應式、無障礙、保留現有配色。完成後用截圖比對設計稿差異並修正。

方案 B：遷移到現代技術棧

HTML/CSS → Vite + React + TypeScript + Tailwind，拆成元件並建立 GitHub Pages 部署。

方案 C：內容與 SEO 強化

替換 Lorem ipsum、加入 Open Graph / JSON-LD、語意化 HTML、效能優化。

方案 D：多頁擴充

新增 /projects、/blog 多頁結構，projects 用 JSON 資料驅動卡片列表。

實戰工作流程

分層用模型（省錢又有效）

探索 / 小改 → Composer / Sonnet
卡住或任務變大 → Fable 5 (high)
極難推理 → Fable 5 (xhigh / max)

effort 等級選擇

  • low 簡單重構、格式整理
  • medium 日常開發任務
  • high 預設，多數 agent 任務
  • xhigh 架構遷移、複雜除錯
  • max 需自我驗證的高風險任務

Cloud Agent 最佳實踐

  • 一次給清楚目標與驗收條件（如「測試通過」「Lighthouse > 90」）
  • 在 User Rules 寫好專案慣例（語言、框架、風格偏好）
  • 長任務用新對話開始，避免舊對話格式錯誤
  • 善用 vision：截圖現有頁面 → 指出問題 → 實作後再截圖比對

成本控管三原則

  1. 簡單任務別預設 Fable 5
  2. 重複的 system prompt / rules 讓 caching 命中
  3. 大任務拆成「規劃（輕模型）」+「執行（Fable 5 Agent）」
